## Newsletter Types

### 1. Curated Newsletter
Collect and comment on the best content, news, and resources in your niche.

| Attribute | Spec |
|-----------|------|
| Best for | Building authority, low content creation burden |
| Frequency | Weekly or daily |
| Length | 500-800 words |
| Structure | 5-10 items with brief commentary (2-3 sentences each) |
| Example format | "3 things I found this week" + commentary |
| Effort | Medium (curation + commentary, not original writing) |

### 2. Original Newsletter
Original insights, frameworks, and analysis. Your unique perspective.

| Attribute | Spec |
|-----------|------|
| Best for | Thought leadership, deep audience connection |
| Frequency | Weekly or biweekly |
| Length | 800-1,500 words |
| Structure | 1-3-1 framework (one insight, three sections, one CTA) |
| Example format | Deep dive on one topic with original analysis |
| Effort | High (original writing every issue) |

### 3. Update/Roundup Newsletter
Company or personal updates, product news, and announcements.

| Attribute | Spec |
|-----------|------|
| Best for | Existing customers, community members |
| Frequency | Weekly, biweekly, or monthly |
| Length | 300-600 words |
| Structure | 3-5 short sections with updates |
| Example format | "What's new this week" + links |
| Effort | Low |

### 4. Mixed Newsletter
Combines original content, curation, and updates. Most common format.

| Attribute | Spec |
|-----------|------|
| Best for | Versatile audience, sustainable publishing |
| Frequency | Weekly |
| Length | 600-1,000 words |
| Structure | 1 original section + 2-3 curated items + 1 CTA |
| Example format | Lead with insight, follow with curated links |
| Effort | Medium |

## Subject Line Formulas

Subject lines determine open rates. These formulas are ordered by effectiveness.

### Formula 1: Specific Number + Benefit
```
"3 frameworks that doubled our engagement"
"7 tools I use to write 10x faster"
"5 mistakes killing your LinkedIn reach"
```

### Formula 2: Curiosity Gap
```
"The metric nobody tracks (but should)"
"Why your best content is invisible"
"What I learned from 100 failed posts"
```

### Formula 3: Direct Benefit
```
"Write LinkedIn hooks in 60 seconds"
"Double your newsletter open rate"
"The template behind 50K-view posts"
```

### Formula 4: Contrarian / Hot Take
```
"Content calendars are a waste of time"
"Stop posting daily"
"Engagement rate is a vanity metric"
```

### Formula 5: Question
```
"Are you making this LinkedIn mistake?"
"What if your content strategy is backwards?"
"Ready to fix your newsletter?"
```

### Formula 6: Personal / Story
```
"I almost quit writing last week"
"The DM that changed my strategy"
"My biggest content failure (and what fixed it)"
```

### Subject line rules
- 5-9 words. 30-50 characters.
- Lowercase preferred (feels personal, less promotional)
- No exclamation marks (spam trigger)
- No ALL CAPS words (spam trigger)
- No misleading claims (destroys trust)
- Test 2 variants (A/B) for every send if audience > 1,000
- Track which formula type performs best for YOUR audience

## Preview Text Strategy

Preview text is the second line visible in the inbox. It complements the subject line.

### Rules
- 40-90 characters (40-50 optimal)
- Never repeat the subject line
- Expand on the subject's promise OR add a second hook
- If no preview text is set, email clients show the first line of the email body -- which is often "View in browser" (bad)

### Patterns

| Subject line | Preview text |
|-------------|-------------|
| "3 frameworks that doubled our engagement" | "Plus: the tool I'm using to track it all." |
| "The metric nobody tracks" | "Hint: it's not likes, shares, or comments." |
| "I almost quit writing last week" | "Then I got this DM." |
| "Stop posting daily" | "Here's the frequency that actually works." |

## Email Structure Patterns

### Pattern 1: The 1-3-1 (Default)
```
[1] Opening hook / thesis (2-3 sentences)

[3] Three supporting sections:
    Section 1: The insight
    Section 2: The evidence or example
    Section 3: The application

[1] CTA (one clear ask)
```

### Pattern 2: The Feature
```
[Hook] One-line hook
[Context] Why this matters (2-3 sentences)
[Deep dive] The main content (400-800 words)
[Key takeaway] One sentence summary
[CTA] What to do next
```

### Pattern 3: The Roundup
```
[Intro] 1-2 sentence theme intro
[Item 1] Link + 2-3 sentence commentary
[Item 2] Link + 2-3 sentence commentary
[Item 3] Link + 2-3 sentence commentary
[Item 4] Link + 2-3 sentence commentary
[Item 5] Link + 2-3 sentence commentary
[CTA] Forward to a friend / reply with your pick
```

### Pattern 4: The Q&A
```
[Intro] This week's question from a reader
[Question] Reader's actual question
[Answer] Your detailed response (300-600 words)
[Bonus] Related resource or tip
[CTA] Submit your question
```

### Pattern 5: The Behind-the-Scenes
```
[Hook] What happened this week
[The story] Transparent look at process, decisions, results
[Numbers] Actual metrics and data
[Lesson] What you learned
[CTA] What you'd like feedback on
```

## Growth Tactics

### 1. Cross-Promotion
- Swap recommendations with newsletters of similar size and audience
- Beehiiv and ConvertKit both support native recommendation features
- Target newsletters with 50-200% of your subscriber count (too large won't reciprocate)
- Track conversion rate per cross-promo partner

### 2. Lead Magnets
- Offer a free resource in exchange for email signup
- Best lead magnets: templates, checklists, frameworks, swipe files, data reports
- Gate the resource behind email capture (landing page or pop-up)
- Deliver via automated welcome email
- Lead magnet quality determines subscriber quality -- don't give away junk

### 3. Referral Program
- Reward subscribers for referring others
- Beehiiv has built-in referral tracking
- Reward tiers: 1 referral = bonus content, 3 = template pack, 10 = 1:1 call
- Mention the referral program in every issue (footer or dedicated section)

### 4. Social-to-Newsletter Funnel
- Every social post should mention the newsletter 1-2x per week (not every post)
- Pin a tweet/post about the newsletter
- LinkedIn featured section should include newsletter link
- Instagram link-in-bio should include newsletter
- Use teaser content: share 80% of an insight on social, full version in newsletter

### 5. Content Upgrades
- Offer bonus content related to a specific post in exchange for email
- "I wrote a full breakdown of this framework -- link in comments (newsletter)"
- Works especially well with carousel posts and threads

### 6. Guest Contributions
- Write guest issues for larger newsletters
- Invite guests to write for yours (brings their audience)
- Co-create content with complementary creators

### 7. Paid Growth
- Meta ads to newsletter landing page ($3-10 per subscriber depending on niche)
- Twitter/X promoted posts about newsletter
- Sponsor other newsletters to promote yours
- Budget: start at $10-20/day, scale what converts

## Deliverability Basics

### Authentication (required)
- **SPF record:** Authorizes your sending server
- **DKIM:** Cryptographically signs your emails
- **DMARC:** Tells receiving servers what to do with unauthenticated email
- All three must be configured for reliable delivery

### Sender reputation
- Send consistently (sudden volume spikes trigger spam filters)
- Clean your list quarterly (remove inactive subscribers)
- Monitor bounce rate (keep under 2%)
- Monitor spam complaints (keep under 0.1%)
- Use a dedicated sending domain (not your main domain)

### Content best practices
- Keep total email size under 102 KB (Gmail clips above this)
- Maintain 60:40 text-to-image ratio minimum
- Include alt text on all images
- Avoid spam trigger words ("free," "guaranteed," "act now")
- Test in multiple clients before sending (Gmail, Outlook, Apple Mail)
- Always include a plain-text version

### List hygiene
- Remove hard bounces immediately
- Remove subscribers inactive for 90+ days (move to re-engagement first)
- Never buy email lists
- Double opt-in is recommended for quality (single opt-in for growth)
- Re-confirmation campaign annually for inactive segments

## Segmentation

### Basic segments (start here)
| Segment | Criteria | Use |
|---------|----------|-----|
| Active | Opened or clicked in last 30 days | Primary audience for all sends |
| Warm | Opened or clicked in last 31-90 days | Re-engagement campaigns |
| Cold | No opens/clicks in 90+ days | Win-back or remove |
| New | Subscribed in last 14 days | Welcome sequence |
| Referrers | Has referred 1+ subscriber | VIP treatment, early access |

### Advanced segments (when audience grows)
| Segment | Criteria | Use |
|---------|----------|-----|
| Topic interest | Clicked on specific topic links | Targeted content |
| Engagement level | High open + click rate consistently | Power users, beta testers |
| Source | Where they subscribed from | Tailor content to acquisition channel |
| Geography | Time zone or location | Send time optimization |

## A/B Testing for Newsletters

### What to test (in priority order)
1. **Subject lines** (biggest impact on opens)
2. **Send time** (day and hour)
3. **Preview text** (second biggest impact on opens)
4. **CTA placement and wording** (biggest impact on clicks)
5. **Email length** (short vs. long)
6. **Content format** (1-3-1 vs. roundup vs. feature)

### Testing rules
- Test ONE variable at a time
- Minimum sample size: 500 per variant (1,000 total)
- Wait 2-4 hours before declaring a winner (early openers skew results)
- Run the same test 2-3 times before concluding (one test is not statistically significant)
- Document every test and result in `logs/newsletter-tests.md`

### Test structure
```
Test: [what you're testing]
Variant A: [description]
Variant B: [description]
Sample: [N] per variant
Metric: [open rate / click rate / reply rate]
Duration: [hours before declaring winner]
Result: [winner + margin]
Action: [what changes based on this]
```

## Newsletter Calendar

### Weekly newsletter template
| Day | Action |
|-----|--------|
| Monday | Collect content ideas, review social performance for newsletter topics |
| Tuesday | Write first draft |
| Wednesday | Edit, add links, design |
| Thursday AM | Final review, A/B test setup, schedule |
| Thursday 8-10 AM | Send |
| Thursday PM | Monitor opens, respond to replies |
| Friday | Review performance, log results |

### Biweekly adjustment
Alternate weeks between writing and research/planning. Writing week follows the schedule above. Off-week: research topics, interview sources, build content backlog.
